# Що потрібно встановити перед стартом (Підготовка)

Перш ніж завантажувати репозиторій з проєктом, перевірте, чи є на вашому комп'ютері базові інструменти:

### 1. Visual Studio Code (VS Code)
Це робоча програма (редактор), де будуть змінюватися файли проєкту.
**Де завантажити:** [code.visualstudio.com](https://code.visualstudio.com/).

*\*Можна використовувати зручні для вас аналоги по типу Cursor.*

### 2. Node.js
Необхідний для запуску нашого сайту.
**Як перевірити:**
1. Натисніть кнопку/меню Пуск або Пошук, введіть `cmd` і відкрийте "командний рядок" з правами адміністратора.
2. В терміналі напишіть `node -v` і натисніть `Enter`.
3. Якщо з'явилися цифри (наприклад, `v20.11.1`), усе добре! Якщо пише "команда не розпізнана", Node.js треба встановити.

**Де завантажити:** [nodejs.org](https://nodejs.org/). На сайті натисніть на велику кнопку з написом **LTS (Recommended for most users)**. Під час встановлення просто скрізь натискайте "Next" (Далі).

### 3. GitHub Desktop
Це зручна програма для роботи з файлами проєкту, яка замінює складні для початківців Git-команди.
**Де завантажити:** [desktop.github.com](https://desktop.github.com/). Увійдіть під своїм акаунтом GitHub.

### 4. Git
Перевірте чи встановився Git разом з GitHub Desktop. Git - це система контролю версій, яка дозволить вам працювати з GitHub на своєму комп'ютері.
**Як перевірити:** Знову у вікні "командного рядка" (`cmd`) напишіть `git -v` і натисніть `Enter`. Якщо бачите текст на кшталт “`git version 2.43.0`”, усе готово.
**Де завантажити:** [git-scm.com/downloads](https://git-scm.com/downloads). Виберіть вашу операційну систему (Windows/Mac) та завантажте. Під час встановлення не змінюйте жодних налаштувань — просто натискайте "Next" (Далі) до самого кінця.

> **Важливо:** Якщо ви щойно встановили Node.js та/або Git, обов'язково перезавантажте комп'ютер, щоб система їх "побачила"!

# Крок 1. Як завантажити (склонувати) проєкт

Найпростіший спосіб скопіювати код з GitHub на свій комп'ютер — використати GitHub Desktop.
1. Перейдіть за посиланням на наш репозиторій у браузері.
2. Натисніть на зелену кнопку **Code**.
3. У меню, що випаде, виберіть **Open with GitHub Desktop**.
4. Браузер запитає дозвіл відкрити програму — погодьтеся.
5. Відкриється програма GitHub Desktop. У полі "Local path" виберіть папку на вашому комп'ютері, куди хочете зберегти проєкт, і натисніть синю кнопку **Clone**.
6. Коли завантаження завершиться, прямо в GitHub Desktop натисніть кнопку **Open in Visual Studio Code**, щоб відкрити проєкт для роботи. // *Або відкрийте VS Code і оберіть потрібну папку з проєктом у розділі/вкладці File (згори) чи в Explorer (зліва)*

# Крок 2. Налаштування проєкту

Коли ви клонуєте проєкт, важкі файли з бібліотеками та деякі типи файлів не завантажуються разом із ним. Їх треба додати один раз вручну.

### 1. Встановлення бібліотек
Відкрийте “командний рядок” як раніше через `cmd` у меню Пуск з правами адміна. Сскопіюйте шлях до папки з проєктом. Напишіть в “командному рядку” `cd` та вставте скопійований шлях до папки проєкту через пробіл без лапок, натисніть `Enter`. Потім напишіть команду `npm install` і знову `Enter`.
*\*Зачекайте кілька хвилин, поки завантажаться всі необхідні системні елементи.*

### 2. Створення файлу конфігурації (.env)
Файл з назвою `.env` треба створити у кореневій теці.
1. У лівій панелі файлів (Explorer) VS Code клікніть правою кнопкою миші на порожньому місці (під списком усіх папок) і виберіть **New File**.
2. Назвіть його `.env`.
3. Відкрийте його і вставте наступні рядки:
```env
BACKEND_URL=[https://shopapi.webshining.space/api/v1](https://shopapi.webshining.space/api/v1)
NEXT_PUBLIC_BACKEND_URL=[https://shopapi.webshining.space/api/v1](https://shopapi.webshining.space/api/v1)
```
4. Збережіть файл/зміни (`Ctrl+S`).

# Крок 3. Запуск проєкту

### 1. Синхронізація з GitHub
Ваш код має бути “найсвіжішим”, щоб не зламати роботу інших!

1. Відкрийте GitHub Desktop. Переконайтеся, що поточний репозиторій `client`, а гілка (current branch) — `main`.
2. Натисніть кнопку **Fetch origin**. Якщо замість неї з'явиться синя кнопка **Pull origin** зі стрілочкою вниз — обов'язково натисніть її, щоб завантажити останні зміни від команди.

### 2. Запуск сайту
1. У “командному рядку” `cmd` з правами адміна зайдіть у папку проєкту через `cd` та введіть команду: `npm run dev`
2. Коли побачите повідомлення про готовність, відкрийте браузер і перейдіть за адресою: `http://localhost:3000`. *(Щоб зупинити сервер, клікніть у відкритий термінал і натисніть `Ctrl+C`, а потім `Y`)*

> **Важливо:** Попередній крок №3 треба повторювати завжди перед початком роботи з файлами проєкту для відстеження результатів своїх дій на веб-сайті. Але слід розуміти, що це буде відображення локальних змін.

## Структура проєкту: Де і що лежить?

Щоб не зламати проєкт, важливо знати, за що відповідає кожна папка.

### Ваша робоча зона (де ви будете працювати):
* **`app/`** — основна папка з ієрархією сторінок застосунку та їх дизайнами/леяутами[cite: 4].
  * **Головне правило:** Щоб створити нову сторінку за певним посиланням, треба створити в ієрархії нову папку. [cite_start]У кожній новій папці ОБОВ’ЯЗКОВО повинен бути файл [cite: 10] [cite_start]з назвою `page.tsx` [cite: 11] (+ зазвичай також файл зі стилями `page.css.ts` для цієї сторінки).
  * *\*Назва файлів може бути не “page”, а наприклад однойменна з назвою розділу/папки, до котрої належить сторінка; або “style(s)” для файлів типу `.css.ts` - головне, щоб назви новостворених файлів були логічними та зрозумілими.*
  * [cite_start]Також тут є файли `layout.tsx` — леяут, мейн елементи для всіх сторінок [cite: 5] (повторювані, як от хедер/футер).
* **`components/`** — у компонентах лежать глобальні стилі[cite: 15].
  * [cite_start]**Як ми пишемо стилі:** Для стилів використовуються java script компоненти[cite: 12]. [cite_start]Якщо ви бачите в коді `import styled from "styled-components"`[cite: 13], це означає, що тут задається зовнішній вигляд елемента.
  * *\*Файли папки components та файли layout папки app краще не редагувати без потреби.*

### Усі інші папки – це зона відповідальності Артема (туди краще не лізти)
Ці папки містять складну логіку роботи, їх редагувати НЕ потрібно:
* **`messages/`** — переклади.
* **`store/`** — файли-сховища зі всякою інфою, типу категорій + модалки[cite: 2].
* **`types/`** — типи, котрі «приходять» з бекенду[cite: 3].

# Крок 4. Як вносити свої зміни в проєкт

**Головне правило розробки:** ми НІКОЛИ не вносимо зміни безпосередньо в головну версію сайту (гілка `main`).
Уявіть, що `main` — це чистовик. Щоб щось змінити, ми створюємо "чернетку" (нову гілку), робимо там свою роботу, а потім просимо перевірити її і додати до чистовика. Цей процес запиту на додавання називається **Pull Request**.

Ось як це зробити крок за кроком через GitHub Desktop:

### Етап 1: Створення своєї гілки
1. Відкрийте GitHub Desktop.
2. Переконайтеся, що у вкладці сurrent branch вибрано `main`.
3. Натисніть кнопку **Fetch origin**, щоб підтягнути найсвіжіші зміни від інших учасників команди.
4. Тепер натисніть на сurrent branch і виберіть **new branch**.
5. Назвіть свою гілку своїм ім’ям/ніком зрозумілою англійською мовою (без пробілів, використовуйте дефіси).
6. Натисніть **create branch**. Переконайтесь, що у сurrent branch тепер обрано вашу новостворену гілку.

### Етап 2: Збереження своєї роботи (Коміт)
*\*Попередньо визначіть з Настею та/або Артемом що саме (які задачі) ви будете виконувати, щоб вони не перетинались з роботою інших*

1. Відкрийте проєкт у VS Code, внесіть необхідні зміни (додайте сторінку, змініть текст чи дизайн) і збережіть файли (`Ctrl + S`).
2. Відкрийте GitHub Desktop. Зліва ви побачите список усіх файлів, які ви змінили.
3. У лівому нижньому куті є поле **Summary (required)**. Напишіть туди коротко, що саме ви зробили (наприклад: *"Додав кнопку купити на сторінку товару"* – краще англійською). Можете за бажанням написати детальніший короткий опис нижче.
4. Натисніть синю кнопку **Commit to [назва_вашої_гілки]**. Це збереже ваші зміни в історії гілки.

### Етап 3: Відправка на перевірку (Pull Request)
1. Після коміту у GitHub Desktop з'явиться синя кнопка **Publish branch** (або **Push origin**). Натисніть її, щоб опублікувати вашу гілку на GitHub.
2. Щойно гілка завантажиться, там же з'явиться кнопка **Create Pull Request**. Натисніть її.
3. У вас відкриється сторінка GitHub у браузері.
4. Додайте короткий опис того, що ви зробили, у велике текстове поле і натисніть зелену кнопку **Create pull request**.

### Етап 4: Обов’язкове затвердження (Discord)
Ви створили запит, але він ще не доданий до головного сайту. Щоб ваші зміни потрапили в “чистовик” (`main`), їх повинен перевірити та схвалити головний розробник (Артем).

**Ваші дії після створення Pull Request:**
1. Напишіть Артему/Насті в особисті повідомлення. Домовтеся про короткий дзвінок (зустріч у голосовому каналі Discord) для демонстрації та затвердження результатів вашої роботи.
2. Артем та/або Настя розгляне ваш код разом з вами на дзвінку. Ваші зміни або затвердять одразу (Merge), або пояснять, що треба виправити, і відхилять запит на доопрацювання.

Будь ласка, не ображайтесь на зауваження до коду — це нормальний робочий процес, який робить наш проєкт кращим!
